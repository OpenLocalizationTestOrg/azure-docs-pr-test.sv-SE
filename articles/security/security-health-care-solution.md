# <a name="a-practical-guide-to-designing-secure-health-care-solutions-in-azure"></a>En praktiska guide för att skapa säkra hälsovård lösningar i Azure
Hälsotillstånd branschen omstarter, systemintegrerare (SIs), oberoende programvaruleverantörer (ISV) och vårdorganisationer som överväger en flytt till Azure letar efter riktlinjer som hjälper dem att inkludera säkerhetsåtgärder för att uppfylla skyldigheterna kompatibilitet.

[En praktiska Guide för att designa Secure hälsovård lösningar i Microsoft Azure](https://aka.ms/azureindustrysecurity) hjälper till att du förstår hur du kan förbättra säkerheten för dina lösningar med hjälp av Azure-tjänster och funktioner som du kan konfigurera baserat på dina krav.
Innehållet är uppdelat i tre huvudavsnitt:

1. Överväganden riktlinjer för att använda molnteknik, inklusive riskhantering, dela ansvar, etablera ett hanteringssystem för informationssäkerhet, Förstå branschen och lokala föreskrifter och upprätta standardprocedurer.
2. Viktiga säkerhetsprinciper som både justeras till en standard information security management standard, till exempel ISO 27001, och standard utvecklingsprocesser, till exempel Microsoft Security Development Lifecycle (SDL).
3. Tillämpa viktiga principerna användningsfall av visar justering ur lösning systemarkitekt där kraven för lösningarna som ligger i linje till standard information security management.

Vi hoppas att du hittar [praktiska guiden till designa Secure hälsovård lösningar](https://aka.ms/azureindustrysecurity) användbara och om du har några frågor eller förslag kan berätta för oss genom att lämna en kommentar som nedan.

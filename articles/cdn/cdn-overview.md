## <a name="how-it-works"></a><span data-ttu-id="cb249-113">Hur det fungerar</span><span class="sxs-lookup"><span data-stu-id="cb249-113">How it works</span></span>
![Översikt över CDN](./media/cdn-overview/cdn-overview.png)

1. <span data-ttu-id="cb249-115">En användare (Alice) begär en fil (även kallad tillgång eller resurs) med hjälp av en URL med ett särskilt domännamn, t.ex. `<endpointname>.azureedge.net`.</span><span class="sxs-lookup"><span data-stu-id="cb249-115">A user (Alice) requests a file (also called an asset) using a URL with a special domain name, such as `<endpointname>.azureedge.net`.</span></span>  <span data-ttu-id="cb249-116">DNS omdirigerar begäran till den bästa tillgängliga POP-platsen (Point-of-Presence).</span><span class="sxs-lookup"><span data-stu-id="cb249-116">DNS routes the request to the best performing Point-of-Presence (POP) location.</span></span>  <span data-ttu-id="cb249-117">Detta är vanligtvis den POP-plats som ligger närmast användaren geografiskt sett.</span><span class="sxs-lookup"><span data-stu-id="cb249-117">Usually this is the POP that is geographically closest to the user.</span></span>
2. <span data-ttu-id="cb249-118">Om filen inte finns i cacheminnet på edge-servrarna på POP-platsen begär edge-servern filen från ursprunget.</span><span class="sxs-lookup"><span data-stu-id="cb249-118">If the edge servers in the POP do not have the file in their cache, the edge server requests the file from the origin.</span></span>  <span data-ttu-id="cb249-119">Ursprunget kan vara ett Azure Web Apps-, Azure Cloud Services- eller Azure Storage-konto eller en offentligt tillgänglig webbserver.</span><span class="sxs-lookup"><span data-stu-id="cb249-119">The origin can be an Azure Web App, Azure Cloud Service, Azure Storage account, or any publicly accessible web server.</span></span>
3. <span data-ttu-id="cb249-120">Ursprunget returnerar filen till edge-servern, inklusive valfria HTTP-sidhuvuden som beskriver filens TTL (Time-To-Live).</span><span class="sxs-lookup"><span data-stu-id="cb249-120">The origin returns the file to the edge server, including optional HTTP headers describing the file's Time-to-Live (TTL).</span></span>
4. <span data-ttu-id="cb249-121">Edge-servern cachelagrar filen och returnerar den till användaren som ursprungligen begärde den (Alice).</span><span class="sxs-lookup"><span data-stu-id="cb249-121">The edge server caches the file and returns the file to the original requestor (Alice).</span></span>  <span data-ttu-id="cb249-122">Filen förblir cachelagrad på edge-servern tills TTL-perioden upphör.</span><span class="sxs-lookup"><span data-stu-id="cb249-122">The file remains cached on the edge server until the TTL expires.</span></span>  <span data-ttu-id="cb249-123">Om ursprunget inte angav någon TTL är standardvärdet sju dagar.</span><span class="sxs-lookup"><span data-stu-id="cb249-123">If the origin didn't specify a TTL, the default TTL is seven days.</span></span>
5. <span data-ttu-id="cb249-124">Fler användare kan begära samma fil med samma URL och kan också dirigeras till samma POP.</span><span class="sxs-lookup"><span data-stu-id="cb249-124">Additional users may then request the same file using that same URL, and may also be directed to that same POP.</span></span>
6. <span data-ttu-id="cb249-125">Om filens TTL inte har gått ut returnerar edge-servern filen från cachen.</span><span class="sxs-lookup"><span data-stu-id="cb249-125">If the TTL for the file hasn't expired, the edge server returns the file from the cache.</span></span>  <span data-ttu-id="cb249-126">Detta resulterar i en snabbare och mer responsiv användarupplevelse.</span><span class="sxs-lookup"><span data-stu-id="cb249-126">This results in a faster, more responsive user experience.</span></span>

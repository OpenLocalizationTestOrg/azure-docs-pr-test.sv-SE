## <a name="metaverse"></a><span data-ttu-id="15d20-140">Metaversum</span><span class="sxs-lookup"><span data-stu-id="15d20-140">Metaverse</span></span>
<span data-ttu-id="15d20-141">Metaversum är samlad vy över alla kopplade identiteter från intilliggande kopplingens utrymmen.</span><span class="sxs-lookup"><span data-stu-id="15d20-141">The metaverse is the consolidated view of all joined identities from neighboring connector spaces.</span></span>

<span data-ttu-id="15d20-142">Identiteter är länkade till varandra, och tilldelas behörighet för olika attribut via importera mappningar av attributflöde börjar centrala metaversumobjekt samlar in information från flera system.</span><span class="sxs-lookup"><span data-stu-id="15d20-142">As identities are linked together and authority is assigned for various attributes through import flow mappings, the central metaverse object begins to aggregate information from multiple systems.</span></span> <span data-ttu-id="15d20-143">Från det här objektet attributflöde utföra mappningar information för utgående system.</span><span class="sxs-lookup"><span data-stu-id="15d20-143">From this object attribute flow, mappings carry information to outbound systems.</span></span>

<span data-ttu-id="15d20-146">Objekt i metaversumet kan inte ändras direkt.</span><span class="sxs-lookup"><span data-stu-id="15d20-146">Objects in the metaverse cannot be edited directly.</span></span> <span data-ttu-id="15d20-147">Alla data i objektet måste vara bidragit via attributflöde.</span><span class="sxs-lookup"><span data-stu-id="15d20-147">All data in the object must be contributed through attribute flow.</span></span> <span data-ttu-id="15d20-148">Metaversum underhåller beständiga kopplingar med varje anslutningsplatsen.</span><span class="sxs-lookup"><span data-stu-id="15d20-148">The metaverse maintains persistent connectors with each connector space.</span></span> <span data-ttu-id="15d20-149">Dessa anslutningar kräver inte utvärdering för varje synkronisering körs.</span><span class="sxs-lookup"><span data-stu-id="15d20-149">These connectors do not require reevaluation for each synchronization run.</span></span> <span data-ttu-id="15d20-150">Det innebär att Azure AD Connect-synkronisering inte har att hitta matchande fjärrobjektet varje gång.</span><span class="sxs-lookup"><span data-stu-id="15d20-150">This means that Azure AD Connect sync does not have to locate the matching remote object each time.</span></span> <span data-ttu-id="15d20-151">Detta förhindrar behovet av kostsamma agenter att attribut som vanligtvis är ansvarig för korrelation mellan objekten ska kunna ändras.</span><span class="sxs-lookup"><span data-stu-id="15d20-151">This avoids the need for costly agents to prevent changes to attributes that would normally be responsible for correlating the objects.</span></span>

<span data-ttu-id="15d20-152">När upptäcka nya datakällor som kan ha redan befintliga objekt som måste hanteras, använder Azure AD Connect-synkronisering en process som kallas en join-regel för att utvärdera potentiella kandidater som du vill skapa en länk.</span><span class="sxs-lookup"><span data-stu-id="15d20-152">When discovering new data sources that may have preexisting objects that need to be managed, Azure AD Connect sync uses a process called a join rule to evaluate potential candidates with which to establish a link.</span></span>
<span data-ttu-id="15d20-153">När länken är etablerad utvärderingen försvinner och normal attributflöde kan ske mellan fjärranslutna ansluten datakälla och metaversum.</span><span class="sxs-lookup"><span data-stu-id="15d20-153">Once the link is established, this evaluation does not reoccur and normal attribute flow can occur between the remote connected data source and the metaverse.</span></span>

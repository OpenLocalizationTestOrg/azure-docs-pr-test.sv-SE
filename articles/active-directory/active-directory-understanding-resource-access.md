<span data-ttu-id="81464-106">I oktober 2013 har klassiska Azure-portalen och Service Management API: er integrerat med Azure Active Directory för att kunna innan för att förbättra användarupplevelsen för att hantera åtkomst till Azure-resurser.</span><span class="sxs-lookup"><span data-stu-id="81464-106">In October 2013, the Azure classic portal and Service Management APIs were integrated with Azure Active Directory in order to lay the groundwork for improving the user experience for managing access to Azure resources.</span></span> <span data-ttu-id="81464-107">Azure Active Directory innehåller redan bra funktioner, till exempel Användarhantering, lokalt katalogsynkronisering, multifaktorautentisering och åtkomstkontroll för programmet.</span><span class="sxs-lookup"><span data-stu-id="81464-107">Azure Active Directory already provides great capabilities such as user management, on-premises directory sync, multi-factor authentication, and application access control.</span></span> <span data-ttu-id="81464-108">Naturligtvis finns göras dessa också tillgängligt för att hantera Azure-resurser lineär.</span><span class="sxs-lookup"><span data-stu-id="81464-108">Naturally, these should also be made available for managing Azure resources across-the-board.</span></span>

<span data-ttu-id="81464-109">Åtkomstkontroll i Azure startar ur faktureringsinformation.</span><span class="sxs-lookup"><span data-stu-id="81464-109">Access control in Azure starts from a billing perspective.</span></span> <span data-ttu-id="81464-110">Ägaren till ett Azure-konto som nås genom att besöka den [Azure Accounts Center](https://account.windowsazure.com/subscriptions), är kontot Administratör (AA).</span><span class="sxs-lookup"><span data-stu-id="81464-110">The owner of an Azure account, accessed by visiting the  [Azure Accounts Center](https://account.windowsazure.com/subscriptions), is the Account Administrator (AA).</span></span> <span data-ttu-id="81464-111">Prenumerationer är en behållare för fakturering, men de fungerar som en säkerhetsgräns: varje prenumeration har en Service systemadministratörskontot (SA) som kan lägga till, ta bort och ändra Azure-resurser i den prenumerationen genom att använda den [klassiska Azure-portalen](https://manage.windowsazure.com/).</span><span class="sxs-lookup"><span data-stu-id="81464-111">Subscriptions are a container for billing, but they also act as a security boundary: each subscription has a Service Administrator (SA) who can add, remove, and modify Azure resources in that subscription by using the [Azure classic portal](https://manage.windowsazure.com/).</span></span> <span data-ttu-id="81464-112">Standard-SA för en ny prenumeration är AA, men AA kan ändra SA i Azure Accounts Center.</span><span class="sxs-lookup"><span data-stu-id="81464-112">The default SA of a new subscription is the AA, but the AA can change the SA in the Azure Accounts Center.</span></span>

<span data-ttu-id="81464-114">Prenumerationer har också en association med en katalog.</span><span class="sxs-lookup"><span data-stu-id="81464-114">Subscriptions also have an association with a directory.</span></span> <span data-ttu-id="81464-115">Katalogen definierar en uppsättning användare.</span><span class="sxs-lookup"><span data-stu-id="81464-115">The directory defines a set of users.</span></span> <span data-ttu-id="81464-116">Det kan vara användare från arbetet eller skolan som skapade katalogen eller de kan vara externa användare (det vill säga Microsoft Accounts).</span><span class="sxs-lookup"><span data-stu-id="81464-116">These can be users from the work or school that created the directory or they can be external users (that is, Microsoft Accounts).</span></span> <span data-ttu-id="81464-117">Prenumerationer är tillgängliga för en delmängd av de directory-användare som har tilldelats som tjänsten systemadministratörskontot (SA) eller Medadministratör (CA); Det enda undantaget är att äldre skäl Microsoft Accounts (tidigare Windows Live ID) kan tilldelas som SA eller Certifikatutfärdare utan att vara närvarande i katalogen.</span><span class="sxs-lookup"><span data-stu-id="81464-117">Subscriptions are accessible by a subset of those directory users who have been assigned as either Service Administrator (SA) or Co-Administrator (CA); the only exception is that, for legacy reasons, Microsoft Accounts (formerly Windows Live ID) can be assigned as SA or CA without being present in the directory.</span></span>

<br><br>![Åtkomstkontroll i Azure][2]

<span data-ttu-id="81464-119">Funktioner i den klassiska Azure-portalen kan SA som är inloggad med ett Account för att ändra den katalogen som en prenumeration är kopplad till med hjälp av den **redigera katalog** på den **prenumerationer** sidan i **inställningar**.</span><span class="sxs-lookup"><span data-stu-id="81464-119">Functionality within the Azure classic portal enables SAs that are signed in using a Microsoft Account to change the directory that a subscription is associated with by using the **Edit Directory** command on the **Subscriptions** page in **Settings**.</span></span> <span data-ttu-id="81464-120">Observera att den här åtgärden har effekter på åtkomstkontroll i den aktuella prenumerationen.</span><span class="sxs-lookup"><span data-stu-id="81464-120">Note that this operation has implications on the access control of that subscription.</span></span>

> [!NOTE]
> <span data-ttu-id="81464-121">Den **redigera katalog** kommandot i den klassiska Azure-portalen är inte tillgänglig för användare som är inloggad med ett arbets- eller Skol-kontot eftersom dessa konton kan logga in endast till den katalog som de tillhör.</span><span class="sxs-lookup"><span data-stu-id="81464-121">The **Edit Directory** command in the Azure classic portal is not available to users who are signed in using a work or school account because those accounts can sign in only to the directory to which they belong.</span></span>

<span data-ttu-id="81464-123">Vid en enkel, en organisation (till exempel Contoso) genomdriva fakturering och åtkomstkontroll på samma uppsättning prenumerationer.</span><span class="sxs-lookup"><span data-stu-id="81464-123">In the simple case, an organization (such as Contoso) will enforce billing and access control across the same set of subscriptions.</span></span> <span data-ttu-id="81464-124">Katalogen är kopplad till prenumerationer som ägs av en enda Azure-konto.</span><span class="sxs-lookup"><span data-stu-id="81464-124">That is, the directory is associated to subscriptions that are owned by a single Azure Account.</span></span> <span data-ttu-id="81464-125">Användarna ser två uppsättningar resurser (visas med orange färg i den föregående bilden) efter genomförd inloggning till den klassiska Azure-portalen:</span><span class="sxs-lookup"><span data-stu-id="81464-125">Upon successful login to the Azure classic portal, users see two collections of resources (depicted in orange in the previous illustration):</span></span>

* <span data-ttu-id="81464-126">Kataloger där användarkontot finns (källkod eller läggas till som ett huvudnamn för sekundärnyckel).</span><span class="sxs-lookup"><span data-stu-id="81464-126">Directories where their user account exists (sourced or added as a foreign principal).</span></span> <span data-ttu-id="81464-127">Observera att katalogen som används för inloggning inte är relevanta för den här beräkningen så att dina kataloger visas alltid oavsett om du har loggat in.</span><span class="sxs-lookup"><span data-stu-id="81464-127">Note that the directory used for login isn’t relevant to this computation, so your directories will always be shown regardless of where you logged in.</span></span>
* <span data-ttu-id="81464-128">Resurser som är en del av prenumerationer som är kopplade till den katalog som används för inloggning och användaren kan komma åt (där de är en SA eller CA).</span><span class="sxs-lookup"><span data-stu-id="81464-128">Resources that are part of subscriptions that are associated with the directory used for login AND which the user can access (where they are an SA or CA).</span></span>

<br><br>![Användare med flera prenumerationer och kataloger][4]

<span data-ttu-id="81464-130">Användare med prenumerationer i flera kataloger har möjlighet att växla aktuell kontext för den klassiska Azure-portalen genom att använda prenumerationsfiltret.</span><span class="sxs-lookup"><span data-stu-id="81464-130">Users with subscriptions across multiple directories have the ability to switch the current context of the Azure classic portal by using the subscription filter.</span></span> <span data-ttu-id="81464-131">Detta resulterar i en separat inloggning till en annan katalog under försättsbladen, men detta kan åstadkommas sömlöst med enkel inloggning (SSO).</span><span class="sxs-lookup"><span data-stu-id="81464-131">Under the covers, this results in a separate login to a different directory, but this is accomplished seamlessly using single sign-on (SSO).</span></span>

<span data-ttu-id="81464-132">Åtgärder som till exempel flytta resurser mellan prenumerationer kan vara svårare på grund av den här vyn för en katalog över prenumerationer.</span><span class="sxs-lookup"><span data-stu-id="81464-132">Operations such as moving resources between subscriptions can be more difficult as a result of this single directory view of subscriptions.</span></span> <span data-ttu-id="81464-133">För att genomföra överföringen resurs, kan det vara nödvändigt att första gången den **redigera katalog** på sidan prenumerationer i **inställningar** associera prenumerationer i samma katalog.</span><span class="sxs-lookup"><span data-stu-id="81464-133">To perform the resource transfer, it may be necessary to first use the **Edit Directory** command on the Subscriptions page in **Settings** to associate the subscriptions to the same directory.</span></span>

## <a name="partitions-in-azure-storage"></a><span data-ttu-id="dd978-125">Partitioner i Azure-lagring</span><span class="sxs-lookup"><span data-stu-id="dd978-125">Partitions in Azure Storage</span></span>
<span data-ttu-id="dd978-126">Alla objekt som innehåller data som lagras i Azure Storage (BLOB, meddelanden, enheter och filer) tillhör en partition och identifieras av en partitionsnyckel.</span><span class="sxs-lookup"><span data-stu-id="dd978-126">Every object that holds data that is stored in Azure Storage (blobs, messages, entities, and files) belongs to a partition, and is identified by a partition key.</span></span> <span data-ttu-id="dd978-127">Partitionen avgör hur Azure Storage belastningsutjämning blobbar, meddelanden, enheter och filer över servrar så att den passar trafik till dessa objekt.</span><span class="sxs-lookup"><span data-stu-id="dd978-127">The partition determines how Azure Storage load balances blobs, messages, entities, and files across servers to meet the traffic needs of those objects.</span></span> <span data-ttu-id="dd978-128">Partitionsnyckeln är unikt och används för att leta upp en blob, meddelande eller enhet.</span><span class="sxs-lookup"><span data-stu-id="dd978-128">The partition key is unique and is used to locate a blob, message, or entity.</span></span>

<span data-ttu-id="dd978-129">Tabellen ovan i [skalbarhetsmål för Standard Lagringskonton](#standard-storage-accounts) visar prestandamål för en enskild partition för varje tjänst.</span><span class="sxs-lookup"><span data-stu-id="dd978-129">The table shown above in [Scalability Targets for Standard Storage Accounts](#standard-storage-accounts) lists the performance targets for a single partition for each service.</span></span>

<span data-ttu-id="dd978-130">Partitioner påverkar belastningsutjämning och skalbarhet för var och en av lagringstjänsterna på följande sätt:</span><span class="sxs-lookup"><span data-stu-id="dd978-130">Partitions affect load balancing and scalability for each of the storage services in the following ways:</span></span>

* <span data-ttu-id="dd978-131">**Blobbar**: Partitionsnyckeln för en blob är kontonamnet behållarnamn + blobbnamnet.</span><span class="sxs-lookup"><span data-stu-id="dd978-131">**Blobs**: The partition key for a blob is account name + container name + blob name.</span></span> <span data-ttu-id="dd978-132">Detta innebär att varje blob kan ha en egen partition om belastningen på blob begär den.</span><span class="sxs-lookup"><span data-stu-id="dd978-132">This means that each blob can have its own partition if load on the blob demands it.</span></span> <span data-ttu-id="dd978-133">Blobbar kan distribueras över flera servrar för att skala ut åtkomst till dem, men en enda blob kan bara hanteras av en enskild server.</span><span class="sxs-lookup"><span data-stu-id="dd978-133">Blobs can be distributed across many servers in order to scale out access to them, but a single blob can only be served by a single server.</span></span> <span data-ttu-id="dd978-134">Blobbar kan samlas i blob-behållare, finns men det inga partitionering effekter från den här grupperingen.</span><span class="sxs-lookup"><span data-stu-id="dd978-134">While blobs can be logically grouped in blob containers, there are no partitioning implications from this grouping.</span></span>
* <span data-ttu-id="dd978-135">**Filer**: Partitionsnyckeln för en fil är namnet + filen resursnamn-konto.</span><span class="sxs-lookup"><span data-stu-id="dd978-135">**Files**: The partition key for a file is account name + file share name.</span></span> <span data-ttu-id="dd978-136">Det innebär att alla filer i en filresurs finns också i en enda partition.</span><span class="sxs-lookup"><span data-stu-id="dd978-136">This means all files in a file share are also in a single partition.</span></span>
* <span data-ttu-id="dd978-137">**Meddelanden**: Partitionsnyckeln för ett meddelande är kontonamnet + könamn, så att alla meddelanden i en kö är grupperade i en enda partition och hanteras av en enskild server.</span><span class="sxs-lookup"><span data-stu-id="dd978-137">**Messages**: The partition key for a message is the account name + queue name, so all messages in a queue are grouped into a single partition and are served by a single server.</span></span> <span data-ttu-id="dd978-138">Olika köer kan bearbetas av olika servrar för belastningsutjämning för men många köer som kan ha ett lagringskonto.</span><span class="sxs-lookup"><span data-stu-id="dd978-138">Different queues may be processed by different servers to balance the load for however many queues a storage account may have.</span></span>
* <span data-ttu-id="dd978-139">**Entiteter**: Partitionsnyckeln för en entitet är kontot namn + tabellnamn + partitions-nyckel, där Partitionsnyckeln är värdet för det obligatoriska användardefinierade **PartitionKey** -egenskapen för entiteten.</span><span class="sxs-lookup"><span data-stu-id="dd978-139">**Entities**: The partition key for an entity is account name + table name + partition key, where the partition key is the value of the required user-defined **PartitionKey** property for the entity.</span></span> <span data-ttu-id="dd978-140">Alla entiteter med samma partitionsnyckelvärde grupperas i samma partition och hanteras av samma partition server.</span><span class="sxs-lookup"><span data-stu-id="dd978-140">All entities with the same partition key value are grouped into the same partition and are served by the same partition server.</span></span> <span data-ttu-id="dd978-141">Det här är en viktig aspekt att förstå vid utformning av ditt program.</span><span class="sxs-lookup"><span data-stu-id="dd978-141">This is an important point to understand in designing your application.</span></span> <span data-ttu-id="dd978-142">Ditt program bör balansera skalbarhet fördelarna med att sprida entiteter över flera partitioner med data access fördelarna med att gruppera enheter i en enda partition.</span><span class="sxs-lookup"><span data-stu-id="dd978-142">Your application should balance the scalability benefits of spreading entities across multiple partitions with the data access advantages of grouping entities in a single partition.</span></span>  

<span data-ttu-id="dd978-143">En stor fördel att gruppera en uppsättning enheter i en tabell i en enda partition är att det är möjligt att utföra batchåtgärder atomiska via entiteter i samma partition, eftersom det finns en partition på en enskild server.</span><span class="sxs-lookup"><span data-stu-id="dd978-143">A key advantage to grouping a set of entities in a table into a single partition is that it's possible to perform atomic batch operations across entities in the same partition, since a partition exists on a single server.</span></span> <span data-ttu-id="dd978-144">Om du vill utföra batchåtgärder på en grupp av enheter, bör du gruppera dem med samma partitionsnyckel.</span><span class="sxs-lookup"><span data-stu-id="dd978-144">Therefore, if you wish to perform batch operations on a group of entities, consider grouping them with the same partition key.</span></span> 

<span data-ttu-id="dd978-145">Å andra sidan enheter som är i samma tabell, men de har olika partitionsnycklar kan vara belastningsutjämnade via olika servrar, vilket gör det möjligt att ha bättre skalbarhet.</span><span class="sxs-lookup"><span data-stu-id="dd978-145">On the other hand, entities that are in the same table but have different partition keys can be load balanced across different servers, making it possible to have greater scalability.</span></span>

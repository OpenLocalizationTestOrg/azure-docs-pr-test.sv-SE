## <a name="cloudyn-itfm-use-cases-enabled-by-usage-and-ratecard-api-integration"></a><span data-ttu-id="8332c-120">Cloudyn ITFM användningsfall aktiveras med användnings- och RateCard API-integrering</span><span class="sxs-lookup"><span data-stu-id="8332c-120">Cloudyn ITFM use cases enabled by Usage and RateCard API integration</span></span>
<span data-ttu-id="8332c-121">Vanliga Cloudyn ITFM användningsfall aktiveras med användning och RateCard APIs inkluderar:</span><span class="sxs-lookup"><span data-stu-id="8332c-121">Common Cloudyn ITFM use cases enabled by usage and RateCard APIs include:</span></span>

* <span data-ttu-id="8332c-122">**Kostnad Analysis** -tillåter molnet kostnader fördelas någon inbyggd identifierande dimension (provider, tjänst, konto, region osv.).</span><span class="sxs-lookup"><span data-stu-id="8332c-122">**Cost Analysis** - Allows cloud costs to be broken down to any native identifying dimension (provider, service, account, region etc.).</span></span> <span data-ttu-id="8332c-123">Azure-användning och RateCard APIs gör detta lättaste genom att tillhandahålla den mest detaljerade uppdelningen av användning och kostnaden data per konto, vilket sedan grupperade och filtreras efter Cloudyn och visas för användaren i en grafisk eller tabell.</span><span class="sxs-lookup"><span data-stu-id="8332c-123">The Azure Usage and RateCard APIs make this an easy task, by providing the most granular breakdown of usage and cost data per account, which is then grouped and filtered by Cloudyn and presented to the user, in a graphic or tabular form.</span></span>

* <span data-ttu-id="8332c-125">**Kostnad allokering 360** -aktiverar ekonomi och IT-chefer att upptäcka verklig kostnadsuppdelning, drivrutiner och trender för deras distribution.</span><span class="sxs-lookup"><span data-stu-id="8332c-125">**Cost Allocation 360** - Enables finance and IT managers to uncover the actual cost breakdown, drivers and trends of their cloud deployment.</span></span> <span data-ttu-id="8332c-126">Det gör att ytterligare hanterare för att associera distribution utgifter med affärsenheter, avdelningar, regioner och mer att tillhandahålla en oöverträffad insikter om molnet kostnader och underlätta enterprise Återdebiteringar och showbacks.</span><span class="sxs-lookup"><span data-stu-id="8332c-126">It further allows managers to easily associate deployment expenses with business units, departments, regions, and more, providing unprecedented insights into cloud costs, and facilitating enterprise chargebacks and showbacks.</span></span> <span data-ttu-id="8332c-127">Azure-användning och RateCard APIs fungera som indata till Cloudyns kostnaden allokering motorn som kompletterar API: erna genom att definiera metoder och affärslogik för tilldelning av ej taggade eller untaggable resurser.</span><span class="sxs-lookup"><span data-stu-id="8332c-127">The Azure Usage and RateCard APIs serve as input to Cloudyn’s cost allocation engine, which complements the APIs by defining methods and business logic for allocating untagged or untaggable resources.</span></span>

![Kostnad allokering 360-diagram][4]

* <span data-ttu-id="8332c-129">**Kostnadseffektiv storlek** -rekommendationer underutnyttjade virtuella datorer, vilket minskar kundens utgifter i stora eller över etablerade datorer med rätt storlek.</span><span class="sxs-lookup"><span data-stu-id="8332c-129">**Cost-Effective Sizing** - Provides right-sizing recommendations for underutilized virtual machines, thus reducing the customer’s expenses on oversized or over-provisioned machines.</span></span> <span data-ttu-id="8332c-130">Den gör detta genom att undersöka virtuell processor och RAM mått (via prestanda API), timmars körning (via användning API) och kostnaden (via RateCard API).</span><span class="sxs-lookup"><span data-stu-id="8332c-130">It does so by examining virtual machine CPU and RAM metrics (via Performance API), hours of run-time (via Usage API) and cost (via RateCard API).</span></span> <span data-ttu-id="8332c-131">Cloudyn sedan innehåller rätt storlek för rekommendationer baserat på underutnyttjade CPU eller minne resurser (prestanda) och beräknar uppskattade besparingar genom att multiplicera pris delta (RateCard) mellan de virtuella datorerna av den faktiska tid-användningen (användning) av den underutnyttjade datorn.</span><span class="sxs-lookup"><span data-stu-id="8332c-131">Cloudyn then provides right-sizing recommendations based on underutilized CPU or RAM resources (Performance), and calculates estimated savings by multiplying the price delta (RateCard) between the VMs by the actual time-utilization (Usage) of the underutilized machine.</span></span>

![Kostnadseffektiv storlek][5]

* <span data-ttu-id="8332c-133">**Molnet portera rekommendationer** -innehåller finansiella råd om molnet portera.</span><span class="sxs-lookup"><span data-stu-id="8332c-133">**Cloud Porting Recommendations** - Provides financial advice on cloud porting.</span></span> <span data-ttu-id="8332c-134">Den undersöker en användares aktuella kostnaderna för molnresurser som har distribuerats på större molnet leverantörer och jämför den med kostnaden för en motsvarande distribution på Azure.</span><span class="sxs-lookup"><span data-stu-id="8332c-134">It examines a user's current costs of cloud resources which are deployed on major cloud vendors, and compares it to the cost of an equivalent deployment on Azure.</span></span> <span data-ttu-id="8332c-135">Den ger detaljerade, per-resurs, ekonomiskt-baserade portera rekommendationer till Azure.</span><span class="sxs-lookup"><span data-stu-id="8332c-135">It then provides granular, per-resource, financially-based porting recommendations to Azure.</span></span> <span data-ttu-id="8332c-136">Efter en bedömning av motsvarande distributionen krävs i Azure (baserat på mått och användaren prestandainställningar) använder Cloudyn RateCard API för att utvärdera kostnaden för motsvarande distributionen på Azure.</span><span class="sxs-lookup"><span data-stu-id="8332c-136">After assessing the equivalent deployment required on Azure (based on performance metrics and user preferences), Cloudyn uses the RateCard API to evaluate the cost of the equivalent deployment on Azure.</span></span>
* <span data-ttu-id="8332c-137">**Prestandarapporterna** -aktiveras med Azures prestanda API rapporterna innehåller en uppsättning funktioner från processor-och RAM optimering rekommendationer.</span><span class="sxs-lookup"><span data-stu-id="8332c-137">**Performance Reports** - Enabled by Azure’s performance API, these reports provide an array of features from CPU and RAM utilization to optimization recommendations.</span></span> <span data-ttu-id="8332c-138">Nedan visas instansen användning rapporten exempelvis presentera instans uppdelning av Genomsnittlig CPU-användning.</span><span class="sxs-lookup"><span data-stu-id="8332c-138">Below is an instance utilization report example, presenting instance breakdown by average CPU utilization.</span></span>

![Prestandarapporter][6]

* <span data-ttu-id="8332c-140">**Kategori manager** -en kraftfull funktion i Cloudyn som ger ordning för relativt oorganiserad molnresurser.</span><span class="sxs-lookup"><span data-stu-id="8332c-140">**Category manager** - A powerful feature in Cloudyn that brings order to unorganized cloud resources.</span></span> <span data-ttu-id="8332c-141">Den ger användarna friheten att skapa sina egna unika kategorier (taggar) för effektiva mätning och rapportering som är i enlighet med organisationens rutiner.</span><span class="sxs-lookup"><span data-stu-id="8332c-141">It provides users the freedom to create their own unique categories (tags) for effective measuring and reporting that is in line with business practices.</span></span> <span data-ttu-id="8332c-142">Dessutom att användare kan enkelt reglera och kategorisera inkonsekvent Taggning (d.v.s. skrivfel och andra avvikelser) och ej taggade resurser för korrekt kostnad information identifieras automatiskt.</span><span class="sxs-lookup"><span data-stu-id="8332c-142">Further, users can easily regulate and categorize inconsistent tagging (i.e. typos and other discrepancies) and automatically detect untagged resources for accurate cost attribution.</span></span>
